# 任务 15 实现总结：前端表单验证逻辑

## 概述

成功实现了双向交易策略的前端表单验证逻辑，确保用户在提交回测请求前配置了有效的策略参数。

## 实现内容

### 1. 更新 `validateForm()` 函数

**文件**: `web/backtest.js`

**主要改进**:

1. **双向开仓条件验证**
   - 验证至少配置了一个方向的开仓条件（做多、做空或旧版）
   - 如果三个方向都未配置，显示错误：`请至少配置一个方向的开仓条件（做多或做空）`

2. **条件完整性验证**
   - 验证做多开仓条件的完整性（指标、运算符、值都不能为空）
   - 验证做空开仓条件的完整性
   - 验证做多平仓条件的完整性
   - 验证做空平仓条件的完整性
   - 验证旧版开仓/平仓条件的完整性（向后兼容）

3. **止盈止损验证**
   - 验证做多止盈百分比为正数（如果配置了）
   - 验证做多止损百分比为正数（如果配置了）
   - 验证做空止盈百分比为正数（如果配置了）
   - 验证做空止损百分比为正数（如果配置了）
   - 验证旧版止盈止损百分比为正数（向后兼容）

4. **向后兼容性**
   - 保留对旧版 `entryConditions` 和 `exitConditions` 的验证
   - 保留对旧版 `takeProfitPct` 和 `stopLossPct` 的验证

### 2. 新增 `validateConditions()` 辅助函数

**功能**: 验证条件列表的完整性

**参数**:
- `conditions`: 条件数组
- `conditionType`: 条件类型名称（用于错误提示）

**验证规则**:
1. 指标不能为空
2. 运算符不能为空
3. 值不能为空

**错误提示格式**: `{条件类型}的第 {索引} 个条件：{字段}不能为空`

例如：`做多开仓条件的第 1 个条件：指标不能为空`

### 3. 创建测试文件

**文件**: `web/test_form_validation.html`

**测试用例** (共 14 个):

1. ✅ 没有配置任何开仓条件 → 应该失败
2. ✅ 仅配置做多开仓条件 → 应该通过
3. ✅ 仅配置做空开仓条件 → 应该通过
4. ✅ 同时配置做多和做空开仓条件 → 应该通过
5. ✅ 做多开仓条件缺少指标 → 应该失败
6. ✅ 做多开仓条件缺少值 → 应该失败
7. ✅ 做多止盈百分比为负数 → 应该失败
8. ✅ 做多止损百分比为零 → 应该失败
9. ✅ 做空止盈百分比为正数 → 应该通过
10. ✅ 做空止损百分比为负数 → 应该失败
11. ✅ 向后兼容 - 使用旧版开仓条件 → 应该通过
12. ✅ 向后兼容 - 旧版止盈百分比为负数 → 应该失败
13. ✅ 多个做多开仓条件，第二个缺少值 → 应该失败
14. ✅ 完整的双向策略配置 → 应该通过

## 技术细节

### 验证流程
1. 验证基本字段（策略名称、日期范围）
2. 收集所有方向的开仓条件
3. 验证至少配置了一个方向
4. 逐个验证每个方向的条件完整性
5. 验证止盈止损百分比的有效性
6. 返回验证结果

### 代码结构
```javascript
validateForm()
├── 基本字段验证
├── 开仓条件验证
│   ├── 收集双向条件
│   ├── 收集旧版条件
│   └── 验证至少一个方向
├── 条件完整性验证
│   ├── validateConditions(longEntryConditions)
│   ├── validateConditions(shortEntryConditions)
│   ├── validateConditions(longExitConditions)
│   ├── validateConditions(shortExitConditions)
│   ├── validateConditions(entryConditions)
│   └── validateConditions(exitConditions)
└── 止盈止损验证
    ├── 做多止盈止损
    ├── 做空止盈止损
    └── 旧版止盈止损
```
