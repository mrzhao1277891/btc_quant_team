# 杠杆功能实现总结

## 功能概述

成功为BTC回测系统添加了合约交易杠杆支持，允许用户使用杠杆进行回测。

## 实现的功能

### 1. 数据模型更新 (`backend/backtest/models.py`)
- ✅ 在 `StrategyConfig` 类中添加了 `leverage` 字段（默认值为1.0，表示现货交易）
- ✅ 更新了 `to_dict()` 和 `from_dict()` 方法以支持杠杆参数的序列化和反序列化

### 2. 回测引擎更新 (`backend/backtest/engine.py`)
- ✅ 修改了 `_open_position()` 方法，实现杠杆计算逻辑：
  - **实际仓位** = 用户设置的仓位大小（position_size_value）
  - **保证金** = 实际仓位 / 杠杆倍数
  - **持仓数量** = 实际仓位 / 开仓价格
- ✅ 资金检查基于保证金而非实际仓位
- ✅ 盈亏计算基于实际仓位大小（含杠杆）

### 3. API更新 (`backend/backtest/api.py`)
- ✅ 在 `BacktestRequest` 模型中添加了 `leverage` 字段
- ✅ 更新了 `convert_request_to_strategy_config()` 函数以传递杠杆参数

### 4. Web UI更新 (`web/backtest.html`)
- ✅ 在"持仓设置"部分添加了杠杆倍数输入字段
- ✅ 支持1-125倍杠杆（符合主流交易所限制）
- ✅ 添加了友好的提示文本："💡 示例：2000本金 × 5倍杠杆 = 10000仓位"

### 5. JavaScript更新 (`web/backtest.js`)
- ✅ 在 `collectFormData()` 函数中添加了杠杆参数收集
- ✅ 默认值为1.0（现货交易）

### 6. CSS样式更新 (`web/backtest.css`)
- ✅ 添加了 `.hint` 和 `.form-hint` 样式类
- ✅ 美化了提示文本的显示效果

## 计算逻辑说明

### 开仓计算
1. **用户输入**：
   - 持仓大小：10000 USDT（实际仓位）
   - 杠杆倍数：5倍
   
2. **系统计算**：
   - 保证金 = 10000 / 5 = 2000 USDT
   - 持仓数量 = 10000 / 50000 = 0.2 BTC
   - 扣除资金 = 2000 USDT（保证金）

### 盈亏计算
1. **价格变动**：
   - 开仓价：50000 USDT
   - 平仓价：55000 USDT
   - 涨幅：10%

2. **盈亏计算**：
   - 盈亏金额 = (55000 - 50000) × 0.2 = 1000 USDT
   - 盈亏百分比 = 1000 / 2000 × 100% = 50%
   - 杠杆放大效果 = 50% / 10% = 5倍 ✓

## 使用说明

### Web UI使用
1. 打开回测页面：http://127.0.0.1:8001/backtest.html
2. 在"持仓设置"部分找到"杠杆倍数"输入框
3. 输入杠杆倍数（1-125倍）
4. 设置持仓大小（这是实际仓位，系统会自动计算所需保证金）
5. 运行回测

### API使用
```json
{
  "strategy_name": "5倍杠杆测试",
  "initial_capital": 2000,
  "position_size": 10000,
  "position_size_type": "fixed",
  "leverage": 5.0,
  ...
}
```

## 注意事项

1. **杠杆倍数范围**：1-125倍（1倍表示现货交易）
2. **资金检查**：系统检查保证金是否足够，而非实际仓位
3. **盈亏计算**：盈亏基于实际仓位大小（含杠杆），收益率基于保证金
4. **风险提示**：杠杆交易会放大收益和亏损，请谨慎使用
