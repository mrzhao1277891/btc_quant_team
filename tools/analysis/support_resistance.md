# 支撑阻力位分析工具 — 逻辑说明

> 对应实现：`tools/analysis/support_resistance.py`  
> 最后更新：2026-04-24

---

## 一、整体流程

```
四个时间框架各自识别位点
    ↓
每个时间框架：技术位 + 动态位 + 斐波那契位
    ↓
加入心理位（跨所有时间框架）
    ↓
相近位点合并（ATR容差）
    ↓
综合评分（三个维度加权）
    ↓
过滤弱位点，输出最终结果
```

---

## 二、四种位点识别方法

### 1. 技术位（摆动高低点）

K线图上肉眼可见的"山峰"和"山谷"。

**识别规则：** 左右各看 N 根K线，某根K线的 high/low 严格大于/小于两侧所有K线，且波动幅度 > 0.3%（过滤噪音）。

**各周期窗口大小：**

| 周期 | 窗口 | 覆盖时间 |
|------|------|---------|
| 4h   | 3    | 左右各 12 小时 |
| 1d   | 5    | 左右各 5 天 |
| 1w   | 4    | 左右各 4 周 |
| 1M   | 3    | 左右各 3 个月 |

**实盘 vs 回测：**
- 实盘模式：最近 N 根K线改用纯左侧确认（因为右侧数据还没有），避免漏掉最新摆动点
- 回测模式：严格左右两侧确认，保证不引入未来数据

**触碰次数统计：** 识别出摆动点后，统计历史上有多少次价格触及该位置（high/low 进入 ±1.5% 范围）并发生反转。触碰次数越多，说明市场对这个位置的"记忆"越强。

---

### 2. 动态位（均线 + 布林带）

均线和布林带会随价格移动，当前价格在某条均线上方 → 该均线是支撑；在下方 → 是阻力。

**使用的指标：**

| 指标 | 基础权重 | 说明 |
|------|---------|------|
| EMA50 | 3 | 长期趋势分界线 |
| 布林中轨 | 3 | SMA20，波段中轴 |
| EMA25 | 2 | 中期趋势 |
| EMA12 | 2 | 短期趋势 |
| EMA7  | 1 | 超短期，噪音较大 |
| 布林上轨 | 2 | 通常是压力 |
| 布林下轨 | 2 | 通常是支撑 |

**触碰验证（关键改进）：**

用每根K线**自己的均线值**（而非当前均线值）做历史比对，避免用现在的均线位置去评判过去的有效性。

判断逻辑：
1. 触及：`high >= 当时均线 - 容差` 且 `low <= 当时均线 + 容差`
2. 反转：收盘价前后方向不同（形成局部高低点）

两个条件都满足才算一次有效触碰，触碰次数加成到权重（最多 +2）。

---

### 3. 斐波那契位

取近期一段明显的涨跌波段，按关键比例画出回撤位。

**使用的比例：** 0.382、0.5、0.618（只保留黄金分割位，减少噪音）

**延伸位：** 1.272、1.618

**波段识别：** 在最近 N 根K线内找局部高低点，判断最近是上涨波段还是下跌波段，计算对应的回撤/延伸位。

---

### 4. 心理位（整数关口）

BTC 市场参与者天然会在整数位挂单，形成支撑阻力。

- 每 1000 刀一个关口（如 $75,000、$76,000）
- 每 5000 刀是更重要的关口（如 $75,000、$80,000）
- 扫描范围：当前价格 ±20%

心理位标记为 `timeframe = 'all'`，适用于所有时间框架，但**不参与多时间框架共振计分**（避免虚高）。

---

## 三、多时间框架融合

对月线、周线、日线、4H 分别识别位点，然后合并。

**时间框架权重（金字塔决策）：**

| 周期 | 权重 | 定位 |
|------|------|------|
| 1M   | 4    | 战略层，定大局 |
| 1w   | 3    | 战役层，定波段 |
| 1d   | 2    | 战术层，定节奏 |
| 4h   | 1    | 执行层，定入场 |

**合并规则：**

用 ATR 作为容差（最大 1.5%），把价格相近的位点合并成一个区域。合并后记录：
- 平均价格
- 来自哪些时间框架（`timeframes` 列表）
- 来自哪些类型（技术位/动态位/斐波那契/心理位）
- 触碰次数累加
- 平均时间框架权重（用于评分）

---

## 四、综合评分（1-15分）

每个合并后的位点计算一个综合分，三个维度加权：

### 维度1：触碰/置信度（权重 50%）

历史验证次数，最能预测位点有效性。

| touch_count | 得分 |
|-------------|------|
| 1次 | 0.30 |
| 2次 | 0.50 |
| 3次 | 0.70 |
| 4次 | 0.90 |
| 5次以上 | 1.00 |

- 如果优化器提供了 `confidence` 值，取 `touch_count` 得分和 `confidence` 的均值（互相印证）
- 没有任何历史验证数据时，默认 0.2（低分）

### 维度2：多时间框架共振（权重 30%）

同一价格区域在几个不同周期都有位点，越多越可靠。

**只统计真实周期（1M/1w/1d/4h），心理位的 `all` 不算。**

| 跨越周期数 | 得分 |
|-----------|------|
| 1个周期 | 0.20 |
| 2个周期 | 0.40 |
| 3个周期 | 0.65 |
| 4个周期 | 1.00 |

### 维度3：时间框架权威性（权重 20%）

用合并组内的**平均权重**（不是最高权重），这样月线+4H融合的位点和纯月线位点能区分开。

| 平均权重 | 得分 |
|---------|------|
| 1（纯4H） | 0.00 |
| 2（日线为主） | 0.33 |
| 3（周线为主） | 0.67 |
| 4（月线为主） | 1.00 |

### 最终得分

```
score = 置信度 × 0.5 + 共振 × 0.3 + 权威性 × 0.2
final_score = round(score × 14) + 1   # 映射到 1~15 分
```

### 强度等级

| 分数 | 等级 | 含义 |
|------|------|------|
| 13-15 | ★★★★★ 极强 | 高置信度，核心交易位 |
| 10-12 | ★★★★ 强 | 可靠交易位 |
| 7-9   | ★★★ 中等 | 参考交易位 |
| 4-6   | ★★ 弱 | 辅助参考，需额外确认 |
| 1-3   | ★ 极弱 | 不建议依赖 |

---

## 五、过滤与输出

- 归一化评分 < 0.2（约等于 3/15 分）的位点被过滤掉
- 支撑位按价格从高到低排列（最近的在前）
- 阻力位按价格从低到高排列（最近的在前）

---

## 六、实盘 vs 回测的差异

| 场景 | 当前价格来源 | 摆动点识别 | 数据截止 |
|------|------------|-----------|---------|
| 实盘 | Binance 实时 ticker，失败降级用 4H 收盘价 | 最近 N 根用左侧确认 | 无限制 |
| 回测 | 历史 4H 收盘价 | 严格左右两侧 | `before_ts` 时间戳之前 |

回测时所有数据库查询都加了 `AND timestamp < before_ts`，严格避免未来数据污染。

---

## 七、评分的局限性

当前评分系统的已知问题：

1. **成交量维度缺失**：成交量确认权重目前为 0，因为没有可靠的历史成交量验证逻辑。有量才有效，这是一个待补充的维度。

2. **斐波那契位的触碰次数**：斐波那契位是计算出来的，没有历史触碰记录，只能用 `base_strength`（0.382/0.618=3，其他=2）作为置信度代理，精度有限。

3. **评分和实际胜率的相关性**：回测显示高分位点（12-13分）的胜率并不总是高于低分位点（10-11分），说明评分系统还有优化空间，特别是在不同市场状态（牛/熊/震荡）下的表现差异。
