# Francis交易档案

## 核心交易参数
- **保证金**: 2403U
- **杠杆**: 5倍
- **单笔最大**: 10000U
- **硬止损**: 100U/笔 (4.2%保证金)
- **持仓时间**: ≤48小时
- **风险要求**: 盈亏比≥2:1
- **分析方法**: 月→周→日→4h金字塔分析

## 计算规则

### 仓位计算
```
最大理论仓位 = 保证金 × 杠杆 = 2403 × 5 = 12,015U
实际可用仓位 = min(最大理论仓位, 单笔最大) = min(12,015U, 10,000U) = 10,000U
```

### 止损计算
```
硬止损金额 = 100U
止损百分比 = 硬止损金额 / 实际仓位 = 100 / 10,000 = 1%
止损价格 = 入场价格 × (1 - 止损百分比)
```

### 目标计算
```
盈亏比要求 = 2:1
目标盈利 = 硬止损金额 × 盈亏比 = 100 × 2 = 200U
目标百分比 = 目标盈利 / 实际仓位 = 200 / 10,000 = 2%
目标价格 = 入场价格 × (1 + 目标百分比)
```

### 做多示例
```
入场价格: $70,000
仓位: 10,000U
止损: $69,300 (100U损失)
目标: $70,200 (200U盈利)
盈亏比: 2:1
风险: 4.2%保证金
```

### 做空示例
```

风险: 4.2%保证金
```

## 交易风格总结
1. **严格风控**: 每笔交易硬止损100U
2. **短期交易**: 持仓不超过48小时
3. **高盈亏比**: 要求≥2:1的风险回报
4. **杠杆使用**: 5倍杠杆，但受单笔最大限制
5. **金字塔分析**: 多时间框架决策流程

## 监控系统适配
- 监控点位设置需考虑100U硬止损
- 告警阈值需基于具体入场价格计算
- 风险管理需严格执行4.2%保证金限制
