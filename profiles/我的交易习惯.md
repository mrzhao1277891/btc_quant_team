# Francis交易档案

## 核心交易参数
- **保证金**: 2403U
- **杠杆**: 5倍
- **单笔最大**: 10000U
- **硬止损**: 200U/笔 (8.3%保证金)
- **最小盈利点数**: 2000点/笔
- **持仓时间**: ≤48小时
- **风险要求**: 盈亏比≥2:1
- **分析方法**: 月→周→日→4h金字塔分析

## 计算规则

### 仓位计算
```
最大理论仓位 = 保证金 × 杠杆 = 2403 × 5 = 12,015U
实际可用仓位 = min(最大理论仓位, 单笔最大) = min(12,015U, 10,000U) = 10,000U
```

### 止损计算
```
硬止损金额 = 200U
止损百分比 = 硬止损金额 / 实际仓位 = 200 / 10,000 = 2%
止损价格 = 入场价格 × (1 - 止损百分比)
```

### 目标计算
```
盈亏比要求 = 2:1
目标盈利 = 硬止损金额 × 盈亏比 = 200 × 2 = 400U
目标百分比 = 目标盈利 / 实际仓位 = 400 / 10,000 = 4%
目标价格 = 入场价格 × (1 + 目标百分比)
最小盈利点数 = 2000点
```


## 交易风格总结
1. **严格风控**: 每笔交易硬止损200U
2. **短期交易**: 持仓不超过48小时
3. **高盈亏比**: 要求≥2:1的风险回报
4. **最小盈利**: 每笔至少2000点
5. **杠杆使用**: 5倍杠杆，但受单笔最大限制
6. **金字塔分析**: 多时间框架决策流程

## 监控系统适配
- 监控点位设置需考虑200U硬止损
- 告警阈值需基于具体入场价格计算
- 风险管理需严格执行8.3%保证金限制
